### AD_RESULT False-Failure Pattern
**Date:** 2026-03-18
**Context:** Goal.md integration mission
**Learning:** Agents that exit with code 0 but fail to emit an `AD_RESULT` marker are recorded as failures by the swarm controller. This caused all 8 completed tasks to appear as failures (100% failure rate) despite all work being verified on HEAD (3212 tests pass, ruff clean, bandit clean). The fix in commit ecd6fa2 ensures workers always emit AD_RESULT markers on task completion.
**Action:** When diagnosing high failure rates, always cross-reference agent exit codes and git log before assuming work was not done. Exit code 0 + no AD_RESULT = tracking bug, not task failure.
